# 上海小学生英语单词学习产品 — 方案讨论

> 本文档用于 Mr王、Miss苗、Miss白 三方协作讨论，避免飞书群聊上下文溢出。

---

## 综合方案（待讨论确认）

### 1. 产品形态：H5 + 小程序双端

- **不做 PWA**，上海家长没有找 PWA 的习惯
- 用 **Taro** 一套代码同时出 H5 和微信小程序
- Phase 1 先做 H5 跑通流程，小程序认证后续再加
- 微信生态是刚需：家长群分享、打卡、订阅消息

### 2. 目标年级：3-5 年级起步

- 3-5 年级竞争虽大，但家长焦虑感更强、转化率更高
- 教材结构更清晰，适合先验证产品模型
- Phase 2 再扩展到 1-2 年级

### 3. 差异化定位

**"沪教版小学英语课本同步背词"**，严格按教材单元编排，学校学到哪单元就练到哪单元。这是竞品都做不到的本地化深度。

### 4. 核心功能

- 卡片背词 + 发音（Web Speech API 起步）
- 艾宾浩斯复习机制
- 拼写模式（听音写词，上海考试有要求）
- 图片联想（卡通配图，小学生视觉记忆）
- 错词本 + 闯关游戏化 + 每日打卡
- 家长端查看学习进度

### 5. 技术选型

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | Taro + React | 一套出 H5 + 小程序 |
| 后端 | FastAPI | 轻量够用 |
| 数据库 | SQLite → PostgreSQL | 起步用 SQLite |
| 语音 | Web Speech API → 科大讯飞 | 先免费方案，跑通后升级 |

### 6. 开发计划（6 周）

| 阶段 | 时间 | 内容 |
|---|---|---|
| Phase 1 | 2.5 周 | 核心：单词卡片、发音、复习曲线、错词本 |
| Phase 2 | 1.5 周 | 闯关游戏化、每日打卡、排行榜 |
| Phase 3 | 1 周 | UI 打磨、埋点、性能优化 |
| Phase 4 | 1 周 | 内测、收集反馈、迭代 |

### 7. 关键风险

1. **教材数据**：沪教版每个年级 4 册书，单元词汇要手动整理，这是最大内容工作量
2. **发音质量**：Web Speech API 是机器音，体验一般但免费
3. **小程序审核**：教育类审核严格，需提前准备资质

## 讨论记录

### 三方观点汇总

**Mr王 的初步思路：**
- 产品形态：优先网页版（PWA）
- 目标年级：3-5 年级
- 核心功能：卡片背词 + 发音、艾宾浩斯复习、闯关游戏化、错词本
- 技术：React + FastAPI + SQLite
- 计划：Phase1 核心 2 周，Phase2 游戏化 1 周，Phase3 打磨 1 周

**Miss苗 的补充建议：**
- 产品形态：建议直接做小程序而非 PWA（上海家长用微信多），Phase 1 先做 H5 跑通
- 目标年级：建议从 1-2 年级起步（低年级家长更舍得花钱，竞争少）
- 差异化：严格按教材单元编排，"上海小学英语课本同步"
- 技术补充：用 Taro 同时出 H5 + 小程序，语音后期接科大讯飞
- 功能补充：拼写模式、图片联想、每日打卡、AI 纠音

**Miss白 的审核意见：**
- 产品形态：PWA 在上海市场是伪需求，小程序获客成本低 → 同意 H5 + 小程序双端
- 目标年级：1-2 年级内容要求极高（配图+配音工作量翻倍），3-5 年级教材结构更清晰 → 建议 3-5 年级起步
- 技术：Taro + FastAPI + SQLite 是合理组合，语音前期用浏览器自带方案
- 开发计划：6 周更合理，4 周太紧
- 关键风险：教材数据整理、发音质量、小程序审核资质
