# 上海小学生英语单词学习产品 — 最终方案

> 约束：3 人团队 / AI 配图免费方案 / 无教育资质 / 上海版教材

---

## 一、产品定位

**"沪教版小学英语同步背词工具"**

- 走工具路线，不做教育机构
- 产品名用"背词/记词/单词卡"类，避开"培训/课堂/学校"
- 用户协议声明"本产品为学习工具，不涉及教育培训服务"
- 小程序类目选**工具**，避开教育资质审核

---

## 二、产品形态

**H5 先行 → 微信小程序**

- Phase 1 只做 H5，不碰小程序审核
- Phase 3 再申请小程序（工具类资质要求低）
- 用 Taro 一套代码，后续转小程序成本极低

---

## 三、目标年级

**Phase 1：三年级上册（沪教版）**

- 约 120-150 词，足够验证完整流程
- 3 年级是上海小学英语分水岭，家长焦虑感开始上升
- 牛津上海版同步做，优先沪教版

---

## 四、核心功能（按优先级）

| 优先级 | 功能 | 说明 |
|---|---|---|
| P0 | 卡片背词 + 发音 | 核心功能，先跑通 |
| P0 | SM-2 复习算法 | 根据记忆程度自动安排复习 |
| P0 | 错词本 | 自动收集，重点复习 |
| P1 | 拼写模式 | 听音写词，上海考试有要求 |
| P1 | 闯关模式 | 按单元分关，1-3 星评价 |
| P1 | 每日打卡 | 连续打卡 + 简单徽章 |
| P2 | AI 配图 | 每个词自动生成卡通图 |
| P2 | 家长端 | 查看学习进度（极简版） |
| P3 | 积分商城 | 虚拟道具兑换 |
| P3 | 小程序版 | 工具类资质要求低 |

## 五、技术选型

| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | Taro + React + Ant Design Mobile | 组件库直接用，不定制设计 |
| 后端 | FastAPI + SQLite | 极简，3 人团队不搞复杂架构 |
| 部署 | Vercel (前端) + Railway (后端) | 免费额度够用 |
| 发音 | Web Speech API | 免费，Phase 1 够用 |
| 配图 | DALL-E / Stable Diffusion 免费 API | AI 生成卡通图 |
| 数据库 | SQLite → PostgreSQL (后期) | 起步不需要数据库 |

## 九、关键风险

| 风险 | 等级 | 对策 |
|---|---|---|
| 教材数据整理量大 | 高 | AI 批量生成初稿 + 人工校对，Phase 1 只做三年级上册 |

| 小程序审核周期长 | 中 | Phase 1-4 只做 H5，不急 |
| 3 人团队产能有限 | 高 | MVP 范围严格限定，砍掉非核心功能 |
